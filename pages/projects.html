WikiMapper is a 3D graph visualisation tool used to explore the links between Wikipedia Pages, written in C++ using OpenGL.

A major emphasis of this project is performance. I have developed realistic imposter shaders for Spheres and Cylinders which have enabled me to render a huge number of nodes and links at high FPS by reducing vertices to 1 for each sphere and 2 for each cylinder.

WikiLoader, a C++ Wikipedia XML file parser project I developed was used extract the data from Wikipedia. The tool exports the pages (graph nodes) and their associated links (graph edges) into a Neo4j database. This program was parallelised which dropped the execution time to process the 100GB XML dump file by 70%.

The above media shows the software interface and demonstration. This project is still a work in progress, and I am currently working on improving the efficiency of the force directed graph and polishing the movement system.

The initial goal of this project was to self host a HA Kubernetes cluster to learn more about Kubernetes. I designed and built a compact rack based off 4 Lenovo Mini PCs. After setting up K3S, I decided that it was more effort than it was worth for a homelab environment, and opted to set up a Proxmox server instead. I hosted a number of services such as a VPN, Home Assistant, and my Spanner WebApp. Now I am working on building a new and improved version of the server.

Above is media of the finished server. The rack was designed in Fusion360 and encases 4 PCs, an integrated switch, and the PC power supplies. The entire enclosure is powered via a single cable.

A gesture-controlled robotic hand, built for my master's dissertation. I designed and 3D printed a hand with 6 degrees of motion. A microcontroller embedded in the hand is sent data from a Raspberry Pi, which receives finger position data via a UDP server from a desktop application. The desktop application utilizes computer vision to track the user's hand.

The desktop application was developed in Qt and implemented Google's machine learning library, MediaPipe, for the hand tracking. This data was translated into servo positions and sent over the internet via a server to the robotic hand. I was able to reduce the total system (including the hand tracking) latency to 320ms.

I had been looking for ways to store our family recipes and access them in an easy way. This recipe site lets me browse, search, and filter recipes by type. Each recipe includes ingredients, instructions, timings, and an image.

The frontend is a vanilla JavaScript SPA that fetches data from a custom PHP API backed by a MySQL database. I also built a password-protected upload portal with a form for manual entry and a URL importer that uses Python's recipe-scrapers library to automatically extract recipe data from supported websites.

Spanner is a website that allows users to analyse their listening habits and playlists. I undertook this project to learn a bit of web development, and as another justification for building my home server. The frontend was developed in Typescript with React. The backend, which communicates with the Spotify API and processes the data, was written in Go.

I hosted the website on my server using Nginx for the routing, however unfortunately, Spotify has now deprecated the track analysis endpoints of the API breaking the entire analysis side of the project, so I no longer host it.

Table Football Robot

At the Nottingham 24 hour Hackathon - HackNotts84, I led a team to win Intel's 1st place prize, building a table football robot and competing against over 40 teams from 25 universities across the UK. This was achieved using computer vision and parts of a 3D printer.

We were able to track the location of the ball in real time using a webcam and OpenCV and send Gcode commands to the robot control board in order to intercept and kick the ball towards the goal.
